# 開発ツールとライブラリの役割

このプロジェクトで使用する主要なツールとライブラリの目的、およびその役割についてまとめます。

## 1. Node.js バージョン管理

### Volta

- **目的**: 開発チーム全体で Node.js のバージョンを統一するため
- **説明**: `package.json` に使用する Node.js のバージョンを記述するだけで、プロジェクトのディレクトリに入った瞬間に自動で指定のバージョンに切り替えてくれるツールです。nvm よりも設定がシンプルで、環境差異による問題を未然に防ぎます。

## 2. UI & スタイリング

### Shadcn/ui

- **目的**: アプリケーションのUIコンポーネント（ボタン、ダイアログ、カード等）を構築するため
- **説明**: 従来のUIライブラリ（MUIなど）とは異なり、必要なコンポーネントをプロジェクトにコピーして使う形式のUIコンポーネント集です。Tailwind CSS をベースにしており、デザインのカスタマイズ性が非常に高いのが特徴です。

### @tanstack/react-table (TanStack Table)

- **目的**: 「試合の組み合わせ設定画面」のような、高機能なテーブル（Excel風のUI）を実装するため
- **説明**: ソート、フィルタリング、編集といった複雑な機能を持つテーブルを構築するためのヘッドレスUIライブラリです。Shadcn/ui の Table コンポーネント（見た目）と組み合わせて使用します。

## 3. フォーム管理

### react-hook-form

- **目的**: 「選手登録フォーム」などのフォーム入力値と送信プロセスを管理するため
- **説明**: フォームの状態管理ライブラリです。不要な再レンダリングを最小限に抑えるためパフォーマンスが高く、Reactにおけるフォーム実装のデファクトスタンダードです。

### zod

- **目的**: フォームの入力値バリデーション（検証）ルールを定義するため
- **説明**: 「代表者名は必須」「メールアドレスの形式であること」といったルールを、TypeScriptと相性の良いスキーマとして定義できます。react-hook-form と組み合わせて、フォーム送信前のデータ検証を行います。

## 4. 状態管理（ステート管理）

### @tanstack/react-query (TanStack Query)

- **目的**: サーバー状態（Firestoreから取得したデータ）を管理するため
- **説明**: Firestoreからのデータ取得、キャッシュ、ローディング状態、エラー処理を自動化してくれるライブラリです。「試合一覧」のデータ取得や「チーム一覧」の管理に使用し、複雑な非同期処理のコードをシンプルにします。

### Zustand

- **目的**: クライアント状態（サーバーとは関係ない、UIのための状態）を管理するため
- **説明**: 非常にシンプルで軽量な状態管理ライブラリです。「モニター操作画面」のタイマーや得点といった、ローカルで完結する状態の管理に使用します。TanStack Query と役割分担することで、クリーンな設計を保ちます。

## 5. コード品質

### Prettier

- **目的**: コードの「見た目」（スタイル）をチーム全体で強制的に統一するため
- **説明**: コードフォーマッターです。開発者がどのようなインデントや改行で書いても、ファイル保存時に自動で統一されたキレイなスタイルに整形し直します。これにより、コードの読みやすさを保ち、スタイルに関する無駄な議論を防ぎます。

## 6. ドキュメントの整備

新しいパッケージを追加したり、既存の設計からズレた場合はドキュメントを整備する。
